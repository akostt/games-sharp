# 🎲 Система управления игровыми сессиями

> Web-приложение для организации и проведения игровых сессий по настольным играм

## 📌 Описание

Система предназначена для управления коллекцией настольных игр, регистрации игроков и организации игровых сессий в игротеках, антикафе и игровых клубах.

## ✨ Основные возможности

- Каталог настольных игр с категориями и издателями
- Управление игроками и их контактами
- Планирование игровых сессий с выбором игры, участников и места
- Управление местами проведения
- Управление игровым оборудованием
- Удобный ввод телефона с автоформатированием +7 (XXX) XXX-XX-XX (поддержка ввода с 8 и +7)
- Ввод и отображение результатов игровых сессий
- Нормализованная структура БД (3НФ) со справочниками статусов/типов/стран

## 🧰 Технологии

- ASP.NET Core MVC 10.0
- Entity Framework Core 10.0
- SQLite (по умолчанию, кроссплатформенно)
- SQL Server (опционально через конфигурацию)
- JavaScript (ES6+)

## 🚀 Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/akostt/games-sharp.git
cd games-sharp

# 2. Восстановить пакеты
dotnet restore

# 3. Применить миграции (для SQLite по умолчанию)
dotnet ef database update

# 4. Запустить приложение
dotnet run
```
Открыть в браузере: http://localhost:5166

### Выбор провайдера БД

- По умолчанию используется SQLite (файл БД создается локально).
- Чтобы использовать SQL Server, установите в appsettings.json:

```json
"Database": {
	"Provider": "sqlserver"
}
```

и настройте строку подключения SqlServerConnection.

## 🗂️ Структура проекта (кратко)

```
Controllers/   # Контроллеры
Models/        # Модели данных
Views/         # Razor представления
Data/          # EF Core контекст
Services/      # Бизнес-сервисы и кеширование справочных данных
Extensions/    # Конфигурация DI, middleware и БД
Helpers/       # Константы и утилиты
wwwroot/       # CSS/JS/статические файлы
```

## 🧱 Нормализация БД (3НФ)

В актуальной схеме строковые повторяющиеся атрибуты вынесены в справочники:

- `Publishers.CountryId -> Countries`
- `Equipments.EquipmentTypeId -> EquipmentTypes`
- `GameSessions.SessionStatusId -> SessionStatuses`

Это устраняет дублирование, уменьшает вероятность ошибок в данных и упрощает дальнейшее расширение системы.
